## 1. Get your API key

Log in to the [HumanStandard dashboard](https://secure.hsverify.com) and go to **Settings → API Keys**. Create a key and copy it — it's shown once.

<Warning>
  Keep your API key secret. Use environment variables, not hardcoded values.
</Warning>

## 2. Analyze a track

POST a publicly accessible audio URL:

<CodeGroup>
```bash cURL
curl -X POST https://app.jobsbyhumans.com/api/analyze \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-cdn.com/track.mp3"}'
```

```python Python
import requests, os

result = requests.post(
    "https://app.jobsbyhumans.com/api/analyze",
    headers={"Authorization": f"Bearer {os.environ['HS_API_KEY']}"},
    json={"url": "https://your-cdn.com/track.mp3"}
).json()

print(result["verdict"])                # "human" | "suspicious"
print(result["tier_verdicts"])          # per-tier breakdown
```

```typescript TypeScript
const result = await fetch("https://app.jobsbyhumans.com/api/analyze", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.HS_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ url: "https://your-cdn.com/track.mp3" }),
}).then(r => r.json());

console.log(result.verdict);
console.log(result.tier_verdicts);
```
</CodeGroup>

## 3. Act on the result

Route based on `tier_verdicts`:

```python
tier = result.get("tier_verdicts") or {}

if tier.get("press_safe") == "suspicious":
    # Reject - first/default press-chain gate
elif tier.get("human_safe") == "suspicious":
    # Reject - distribution-layer gate
elif tier.get("recall") == "suspicious":
    # Send to manual review - borderline
else:
    # Approve
```

<Tip>
  Use `tier_verdicts` for routing, not the top-level `verdict`. Each tier is calibrated to a specific false positive rate. See [Confidence Tiers](/guides/confidence-tiers).
</Tip>
