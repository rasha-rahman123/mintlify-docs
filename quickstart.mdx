## 1. Get your API key

Log in to the [HumanStandard dashboard](https://secure.hsverify.com) and navigate to **Settings → API Keys**. Create a new key and copy it — you won't see it again.

<Warning>
  Keep your API key secret. Do not commit it to source control or expose it in client-side code.
</Warning>

## 2. Submit a track

The simplest way to analyze a track is to POST a publicly accessible audio URL.

<CodeGroup>
```bash cURL
curl -X POST https://app.jobsbyhumans.com/api/v1/analyze \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-cdn.com/track.mp3"}'
```

```python Python
import requests

response = requests.post(
    "https://app.jobsbyhumans.com/api/v1/analyze",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={"url": "https://your-cdn.com/track.mp3"}
)

result = response.json()
print(result["verdict"])                # "human" | "suspicious"
print(result["scores"]["truth_score"])  # 0.0 – 1.0
```

```typescript TypeScript
const response = await fetch("https://app.jobsbyhumans.com/api/v1/analyze", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.HS_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ url: "https://your-cdn.com/track.mp3" }),
});

const result = await response.json();
console.log(result.verdict);
console.log(result.scores.truth_score);
```
</CodeGroup>

## 3. Read the result

```json
{
  "status": "success",
  "verdict": "human",
  "tier_verdicts": {
    "press_safe": null,
    "human_safe": "human",
    "recall": "human"
  },
  "scores": {
    "truth_score": 0.18,
    "geo_score": 0.31,
    "proto_score_v1": 0.29,
    "head_a": 0.14,
    "head_b": 0.22,
    "head_c": 0.19
  },
  "audio": {
    "duration_sec": 214.3,
    "bitrate": 320000,
    "sample_rate": 44100,
    "channels": 2,
    "codec": "mp3"
  }
}
```

The top-level `verdict` is your primary signal. A `truth_score` near 0 indicates strong human characteristics; near 1 indicates strong AI characteristics.

<Tip>
  For production use, rely on `tier_verdicts` rather than the raw `verdict`. The tiers give you calibrated false-positive guarantees. See [Confidence Tiers](/guides/confidence-tiers).
</Tip>
