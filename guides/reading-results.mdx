## The result record

```json
{
  "item_id": "track-001",
  "status": "success",

  "verdict": "suspicious",
  "tier_verdicts": {
    "press_safe": null,
    "human_safe": "suspicious",
    "recall": "suspicious"
  },

  "scores": {
    "confidence_score": 0.87,
    "geometry_score": 0.79,
    "prototype_score": 0.74,
    "spectral_score": 0.91,
    "frequency_artifact_score": 0.83,
    "temporal_consistency_score": 0.77,
    "representation_score": 0.68,
    "reliability": 0.88
  },

  "origin": "suno",
  "origin_confidence": 0.82,
  "calibration_version": "2026-01-12",

  "risk_series": [0.12, 0.14, 0.88, 0.91, 0.89, 0.87],
  "processed_at": "2026-02-27T10:43:01Z"
}
```

## Routing

Use `tier_verdicts` for all routing decisions. It gives you three simultaneous verdicts at different confidence levels:

| Field | Values | Use it for |
|-------|--------|-----------|
| `tier_verdicts.press_safe` | `"human"` / `"suspicious"` / `null` | Press-chain default gate, certification, licensing submissions |
| `tier_verdicts.human_safe` | `"human"` / `"suspicious"` | Distribution-layer auto-reject at intake |
| `tier_verdicts.recall` | `"human"` / `"suspicious"` | Escalation to manual review |

See [Confidence Tiers](/guides/confidence-tiers) for the false positive rate at each tier.

## Key fields

| Field | What it's for |
|-------|--------------|
| `verdict` | Top-level result at your operating tier — `"human"` or `"suspicious"` |
| `confidence_score` | Overall AI probability (0–1). Use for displaying confidence to reviewers |
| `geometry_score` | Multi-signal corroborating score. Stable across audio processing variations |
| `prototype_score` | Score driving the press-safe tier. Include in certification records |
| `risk_series` | AI probability across the track timeline — shows which sections triggered |
| `origin` | Predicted generator: `"suno"`, `"udio"`, `"elevenlabs"`, `"human"`, `"unknown"` |
| `calibration_version` | Model version that produced this result. Include in audit records |
| `reliability` | Result confidence based on audio quality. Below `0.6` — treat with extra caution |

## Routing code

```python Python
def route(record: dict) -> str:
    tier = record.get("tier_verdicts") or {}

    if tier.get("press_safe") == "suspicious":
        return "reject"

    if tier.get("human_safe") == "suspicious":
        return "reject"

    if tier.get("recall") == "suspicious":
        return "manual_review"

    return "approve"
```

```typescript TypeScript
function route(record: Record<string, any>): string {
  const tier = record.tier_verdicts ?? {};

  if (tier.press_safe === "suspicious") return "reject";
  if (tier.human_safe === "suspicious") return "reject";
  if (tier.recall === "suspicious") return "manual_review";
  return "approve";
}
```

## Handling failures

Check `status` before reading `verdict`. On failure, `verdict` is `null` and the `error` / `error_code` fields explain what went wrong:

```json
{
  "item_id": "track-005",
  "status": "failed",
  "verdict": null,
  "error": "Could not download audio: access denied (403). Ensure the URL is publicly accessible.",
  "error_code": "source_forbidden"
}
```

The `error` field is a human-readable message. The `error_code` field is a machine-readable code you can switch on programmatically:

| Code | Meaning |
|------|---------|
| `source_forbidden` | 403 — the URL requires authentication |
| `source_not_found` | 404 — the file was not found at that URL |
| `source_unauthorized` | 401 — the URL requires credentials |
| `timeout` | Analysis timed out |
| `no_workers_available` | GPU capacity temporarily exhausted — retry shortly |
| `audio_decode_error` | The file is not a valid audio format |
| `analysis_error` | Catch-all for other failures |

Always route failed tracks to manual review — never auto-reject on an analysis failure.

<Tip>
  For lightweight integrations, use `?detail=simple` to get a reduced response with the same error fields. See [Simplified Responses](/guides/simplified-responses).
</Tip>
