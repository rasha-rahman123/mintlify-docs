Every result includes three simultaneous verdicts, each calibrated to a different false positive rate. Build policies that route automatically at high confidence and escalate to human review at lower confidence.

Every result includes a `tier_verdicts` object:

```json
"tier_verdicts": {
  "press_safe": null,
  "human_safe": "suspicious",
  "recall": "suspicious"
}
```

## The three tiers

  <Card title="Human-Safe" icon="user-check">

  </Card>
  <Card title="Recall" icon="magnifying-glass">
    **~5–10% false positive rate**

    Manual review queues, audit sampling, secondary screening.
  </Card>
</CardGroup>

## How to read `tier_verdicts`

Each tier value is one of:

| Value | Meaning |
|-------|---------|
| `"suspicious"` | Evidence meets this tier's threshold — call it AI at this confidence level |
| `"human"` | Evidence falls below this tier's threshold — passes at this confidence level |
| `null` | Tier not evaluated (press-safe requires the press-safe model enabled; see below) |

Tiers are independent. A track can be `"suspicious"` at recall but `"human"` at human-safe — that means it's borderline and should go to manual review.

## Typical patterns

```json
// Confident AI detection — flagged at all tiers
{ "press_safe": "suspicious", "human_safe": "suspicious", "recall": "suspicious" }

// Borderline — only caught at recall (escalate to manual review)
{ "press_safe": null, "human_safe": "human", "recall": "suspicious" }

// Clear human — passes all tiers
{ "press_safe": "human", "human_safe": "human", "recall": "human" }

// AI confirmed at human-safe but not press-safe (inconclusive for certification)
{ "press_safe": null, "human_safe": "suspicious", "recall": "suspicious" }
```

## Press-safe and `proto_score_v1`

Press-safe uses a proprietary multi-head acoustic analysis that is stable across the compression artifacts, mastering differences, and loudness normalization that shift score-based thresholds. The underlying score is `proto_score_v1`. Press-safe fires when `proto_score_v1 >= 0.807`.

`press_safe` shows `null` if:
- Press-safe scoring is not enabled for your organization
- The track is too short to score reliably

Contact support to enable press-safe scoring for your account.

## Choosing a tier for your use case

| Use case | Recommended tier | Why |
|----------|-----------------|-----|
| Certification / public attestation | `press_safe` | Near-zero FPR protects reputation |
| Supply chain auto-reject | `human_safe` | Blocks AI confidently, low collateral damage to human artists |
| Escalation to manual review | `recall` | Cast a wider net, let humans decide borderline cases |
| Internal audit / research | All three | Full picture for analysis |

## Performance benchmarks

Measured on the Goal 2 study corpus (450 AI tracks, 150 clean human, 103 hard-negative human):

| Tier | Threshold | AI Detection Rate | FPR (clean human) | FPR (hard negatives) |
|------|-----------|-------------------|-------------------|----------------------|
| Press-safe | 0.807 | 18% | **0%** | **0%** |
| Human-safe | 0.713 | 35% | 2.7% | **0%** |
| Recall | 0.650 | 46% | 8.0% | **0%** |

<Note>
  Hard negatives are human tracks with heavy processing (mastering artifacts, compression, pitch correction) that commonly fool AI detectors. HumanStandard achieves 0% FPR on hard negatives at all tiers.
</Note>
