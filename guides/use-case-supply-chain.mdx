## Tiered escalation

Use all three tiers in a three-layer chain. `press_safe` is the first/default press-chain gate, `human_safe` is the second distribution-layer gate, and `recall` catches borderline cases for human review.

```
Track arrives at intake
          │
          ▼
  Analyze with HumanStandard
          │
          ▼
  ┌─────────────────────────────────────────────┐
  │ Check press_safe tier first (default layer)│
  └─────────────────────────────────────────────┘
          │
          ├── "suspicious" ──► ❌ Auto-reject
          │                      (highest confidence AI, ~0% FPR)
          │
          └── "human"/null ──► Check human_safe tier
                                   │
                                   ├── "suspicious" ──► ❌ Auto-reject
                                   │                      (high confidence AI, ~1-2% FPR)
                                   │
                                   └── "human" ──────► Check recall tier
                                                        │
                                                        ├── "suspicious" ──► 🔍 Manual review queue
                                                        │                      (borderline - human expert decides)
                                                        │
                                                        └── "human" ──────► ✅ Approve
                                                                              (passes all tiers)
```

This gives you four outcomes:
1. **Auto-reject (press chain)** - strongest AI evidence at first/default layer
2. **Auto-reject (distribution layer)** - high-confidence AI at second layer
3. **Manual review** - borderline, needs a human to look
4. **Approve** - passes all tiers, admitted to catalog

## Implementation

```python Python
import requests, os
from enum import Enum

class IntakeDecision(str, Enum):
    APPROVE = "approve"
    MANUAL_REVIEW = "manual_review"
    REJECT = "reject"

def evaluate_intake(audio_url: str, item_id: str) -> dict:
    result = requests.post(
        "https://app.jobsbyhumans.com/api/analyze",
        headers={"Authorization": f"Bearer {os.environ['HS_API_KEY']}"},
        json={"url": audio_url}
    ).json()

    if result.get("status") == "failed":
        # Analysis failed — send to manual review, don't auto-reject
        return {
            "item_id": item_id,
            "decision": IntakeDecision.MANUAL_REVIEW,
            "reason": "analysis_failed",
            "error": result.get("error"),
        }

    tier = result.get("tier_verdicts") or {}
    scores = result.get("scores") or {}

    press_safe = tier.get("press_safe")
    human_safe = tier.get("human_safe")
    recall = tier.get("recall")

    if press_safe == "suspicious":
        # First/default layer (press chain) - strongest AI evidence
        return {
            "item_id": item_id,
            "decision": IntakeDecision.REJECT,
            "reason": "press_safe_suspicious",
            "confidence_score": scores.get("confidence_score"),
            "geometry_score": scores.get("geometry_score"),
            "origin": result.get("origin"),
        }

    if human_safe == "suspicious":
        # Second layer (distribution) - high confidence AI
        return {
            "item_id": item_id,
            "decision": IntakeDecision.REJECT,
            "reason": "human_safe_suspicious",
            "confidence_score": scores.get("confidence_score"),
            "geometry_score": scores.get("geometry_score"),
            "origin": result.get("origin"),
        }

    if recall == "suspicious":
        # Borderline — route to human review
        return {
            "item_id": item_id,
            "decision": IntakeDecision.MANUAL_REVIEW,
            "reason": "recall_suspicious_borderline",
            "confidence_score": scores.get("confidence_score"),
            "geometry_score": scores.get("geometry_score"),
            "risk_series": result.get("risk_series"),
        }

    # Passes all tiers — approve
    return {
        "item_id": item_id,
        "decision": IntakeDecision.APPROVE,
        "confidence_score": scores.get("confidence_score"),
    }
```

```typescript TypeScript
type IntakeDecision = "approve" | "manual_review" | "reject";

interface IntakeResult {
  item_id: string;
  decision: IntakeDecision;
  reason?: string;
  confidence_score?: number | null;
  geometry_score?: number | null;
  origin?: string | null;
  risk_series?: number[];
}

async function evaluateIntake(audioUrl: string, itemId: string): Promise<IntakeResult> {
  const result = await fetch("https://app.jobsbyhumans.com/api/analyze", {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${process.env.HS_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ url: audioUrl }),
  }).then(r => r.json());

  if (result.status === "failed") {
    return { item_id: itemId, decision: "manual_review", reason: "analysis_failed" };
  }

  const tier = result.tier_verdicts ?? {};
  const scores = result.scores ?? {};

  if (tier.press_safe === "suspicious") {
    return {
      item_id: itemId,
      decision: "reject",
      reason: "press_safe_suspicious",
      confidence_score: scores.confidence_score,
      geometry_score: scores.geometry_score,
      origin: result.origin,
    };
  }

  if (tier.human_safe === "suspicious") {
    return {
      item_id: itemId,
      decision: "reject",
      reason: "human_safe_suspicious",
      confidence_score: scores.confidence_score,
      geometry_score: scores.geometry_score,
      origin: result.origin,
    };
  }

  if (tier.recall === "suspicious") {
    return {
      item_id: itemId,
      decision: "manual_review",
      reason: "recall_suspicious_borderline",
      confidence_score: scores.confidence_score,
      geometry_score: scores.geometry_score,
      risk_series: result.risk_series,
    };
  }

  return {
    item_id: itemId,
    decision: "approve",
    confidence_score: scores.confidence_score,
  };
}
```

## What to show human reviewers

When a track lands in manual review, surface the key signals so reviewers can make fast, informed decisions:

| Signal | What to show | What it means |
|--------|-------------|---------------|
| `confidence_score` | Score bar 0–1 | Overall AI probability |
| `risk_series` | Waveform heatmap | Which sections triggered the detector |
| `origin` | Badge | Predicted generator (if any) |
| `spectral_score`, `frequency_artifact_score`, `temporal_consistency_score` | Mini bars | Individual signal scores |
| `geometry_score` | Score | Secondary corroborating score |
| `tier_verdicts.recall` | `"suspicious"` | Why it landed in review |

## Tuning the policy

The three-layer escalation above is a starting point. You can adjust based on your catalog's risk tolerance:

**Stricter (fewer false negatives, more manual review)**:
- Treat `recall == "suspicious"` as auto-reject instead of manual review
- Add a `geometry_score >= 0.5` secondary check before approving

**More permissive (less manual review)**:
- Only auto-reject when `press_safe == "suspicious"` (do not auto-reject on `human_safe`)
- Use `human_safe` to flag for review instead of auto-reject

<Tip>
  Start with the three-layer escalation above, run it for a few weeks, and measure your manual review queue's outcome distribution. If reviewers are overturning too many auto-rejects, raise the threshold. If they're mostly confirming AI, you can safely automate more.
</Tip>
